# Sustainability Notes

DelegationHQ is open source first.

This public repository should focus on a useful local product: a CLI that helps
people plan, approve, record, evaluate, and improve agentic work. It should not
read like a pricing plan or a private company roadmap.

## Public Commitment

The open-source core should stay useful on its own:

- Harnessfile validation
- dry-run planning
- local run ledger
- basic evals
- adapter contracts
- playbooks and examples
- package CLI
- safe GitHub-native workflows

That core is the adoption loop and the trust engine.

## Future Commercial Work

Commercial products or services may exist later, but they should not weaken the
public core. Possible future paid work belongs outside the public repository
until it is ready to announce and support.

Keep the public message simple:

```text
DelegationHQ is mission control for agentic work.
```

See `docs/open-core-strategy.md` for the public/private boundary.
